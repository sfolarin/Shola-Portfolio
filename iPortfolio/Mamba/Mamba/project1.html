Deploy Your Code on Docker Container Using Jenkins on AWS

Overview

In this blog, we are going to deploy a Java Web app on a Docker Container built on an EC2 Instance through the use of Jenkins.

Agenda

  • Setup Jenkins
  • Setup & Configure Maven and Git
  • Integrate GitHub and Maven with Jenkins
  • Setup Docker Host
  • Integrate Docker with Jenkins
  • Automate the Build and Deploy process using Jenkins
  • Test the deployment

Prerequisites

  • AWS Account
  • Git/ GitHub Account with the Source Code
  • A local machine with CLI Access
  • Familiarity with Docker and Git

Step 1: Setup Jenkins Server on AWS EC2 Instance

Setup a Linux EC2 Instance, install Java, Jenkins, and access the Jenkins Web UI on port 8080. Configure EC2 instance settings including VPC, security groups, and key pairs.

Step 2: Integrate GitHub with Jenkins

Install Git and GitHub Plugin on Jenkins, configure Git on Jenkins UI.

Step 3: Integrate Maven with Jenkins

Install Maven on Jenkins Server, setup environment variables, install Maven Plugin, and configure Maven and Java.

Step 4: Setup a Docker Host

Setup a Linux EC2 Instance, install Docker, and start Docker services. Create and run a Tomcat Docker container, and address common issues related to Tomcat Docker images.

Step 5: Integrate Docker with Jenkins

Create a dockeradmin user, install the "Publish Over SSH" plugin, and configure Docker host in Jenkins.

Step 6: Create Jenkins Job to Build and Copy Artifacts on to Docker Host

Create a Jenkins job to build and deploy artifacts to the Docker host. Configure post-build steps to handle artifact transfer and deployment.

Step 7: Update Dockerfile to Copy Artifacts to Launch New Container

Create a Dockerfile to include the webapp.war file, build a new Docker image, and run a new Docker container.

Step 8: Automate Build and Deployment on Docker Container

Automate the Jenkins pipeline to build, create artifacts, and deploy Docker containers automatically upon code commits.

Conclusion

In this blog, we learned how to automate the build and deploy process using GitHub, Jenkins, Docker, and AWS EC2.
