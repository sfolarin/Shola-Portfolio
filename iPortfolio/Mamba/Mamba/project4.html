Fun with Linux for Cloud & DevOps Engineers

Introduction

New to Linux? This assignment covers all the required basics of Linux to be familiar for a DevOps engineer.

Introduction to Linux

Skills

Below skills are required to complete the deployment steps:

  • Linux User Management
  • Permissions
  • Directory Structure
  • File Systems
  • File Management

Pre-Requisites

  • Login to AWS cloud and create a Linux-based EC2 instance to complete the assignment.

Deployment

Login to the server as superuser and perform the following tasks:

  • Create users and set passwords – user1, user2, user3
  • Create Groups – devops, aws
  • Change the primary group of user2 and user3 to devops group
  • Add aws group as a secondary group to user1
  • Create the file and directory structure as shown in the diagram.
  • Change the group of /dir1, /dir7/dir10, /f2 to devops group
  • Change ownership of /dir1, /dir7/dir10, /f2 to user1
  • Login as user1 and perform the following:
    • Create users and set passwords – user4, user5
    • Create Groups – app, database
  • Login as user4 and perform the following:
    • Create directory – /dir6/dir4
    • Create file – /f3
    • Move the file from /dir1/f1 to /dir2/dir1/dir2
    • Rename the file /f2 to /f4
  • Login as user1 and perform the following:
    • Create directory – /home/user2/dir1
    • Change to /dir2/dir1/dir2/dir10 directory and create file /opt/dir14/dir10/f1 using relative path method.
    • Move the file from /opt/dir14/dir10/f1 to user1 home directory
    • Delete the directory recursively /dir4
    • Delete all child files and directories under /opt/dir14 using a single command.
    • Write this text Linux assessment for a DevOps Engineer!! Learn with Fun!! to the /f3 file and save it.
  • Login as user2 and perform the following:
    • Create file /dir1/f2
    • Delete /dir6
    • Delete /dir8
    • Replace the text DevOps with devops in the /f3 file without using an editor.
    • Using Vi-Editor, copy line 1 and paste it 10 times in the file /f3.
    • Search for the pattern Engineer and replace it with engineer in the file /f3 using a single command.
    • Delete /f3
  • Login as root user and perform the following:
    • Search for the file name f3 on the server and list all absolute paths where f3 file is found.
    • Show the count of the number of files in the directory /
    • Print the last line of the file /etc/passwd
    • Login to AWS and create a 5GB EBS volume in the same AZ as the EC2 instance and attach the EBS volume to the instance.
    • Login as root user and perform the following:
      • Create a File System on the new EBS volume attached in the previous step
      • Mount the File System on /data directory
      • Verify File System utilization using df -h command – This command must show /data file system
      • Create file f1 in the /data file system.
  • Login as user5 and perform the following:
    • Delete /dir1
    • Delete /dir2
    • Delete /dir3
    • Delete /dir5
    • Delete /dir7
    • Delete /f1 & /f4
    • Delete /opt/dir14
  • Login as root user and perform the following:
    • Delete users – user1, user2, user3, user4, user5
    • Delete groups – app, aws, database, devops
    • Delete home directories of all users user1, user2, user3, user4, user5 if any exist still.
    • Unmount /data file system
    • Delete /data directory
    • Login to AWS and detach the EBS volume from the EC2 instance and delete the volume, then terminate the EC2 instance.

    All done? Still not confident? Repeat the steps!
